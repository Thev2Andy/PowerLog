About PowerLog

PowerLog is a modular, dependency-free, lightweight .NET logging library made in C#, that is just built different.

It's been built from the ground up with modularity in mind, and not relying on any other dependency, having a different and more compact internal structure. PowerLog is designed in such a way that everything is configurable and easy to use, allowing developers to do complex stuff with only a few lines of code.

    PowerLog has been built from the ground up with modularity in mind, and not relying on any other dependency, having a different and more compact internal structure, which gets it to sort of compete with other logging libraries, just because it was designed this way.

  • Support for the latest .NET.

    PowerLog is currently using .NET 6, and will update to .NET 7 once it comes out.

  • Cross-platform support.

    PowerLog doesn't use any platform-specific APIs, and should run on most platforms. (Tested on Linux Lite)

So what can I do with this?

You can do anything you want, as long as it's within the limitations of your programming language. So no nukes.

On a more serious note, PowerLog is fully customizable, serializable, and raises events everywhere, so you can adapt it to almost any app, and if not, you can contribute to it on GitHub, or compile it from source.

So how do I implement this into my application?

Implementing PowerLog should be pretty straight-forward, you first include PowerLog in your project, obviously, either source code, DLL, or nuget package.

To get a quick example up and running first import the `PowerLog` namespace.
Next up, create a `Log` instance, and log something with it.

You should probably check out the (upcoming) docs though.
